Reading text files¶

We're gonna open and run some external files in this section. If you want run this notebook for yourself, make sure that you are in same directory as the data files and scripts. You can get everything here:

In [1]: 
#go to the directory with with the python demo files.
%cd "/Users/ewilson2011/git/Python_plotting_demo"

/Users/ewilson2011/git/Python_plotting_demo

Obviously, the path to your directory will be different.

Now that we are in the right directory, lets run the demo. There are actually several ways to do this. To read in a simple ascii file (one with just numbers arranged uniformly in columns), you can use:

In [2]: 
import numpy as np
a = np.loadtxt("coastline.txt")

Any valid text file extension (such as .dat) should work. The np.loadtxt reads in the simple text file as a Numpy array. In this example, the coastline.txt file contains two columns of numbers representing the lat/lon coordinates of the coastline. a is therefore a two column array. You could also read the columns of ascii files into separate variables:

In [3]: 
lon,lat = np.loadtxt("coastline.txt", unpack = True)

This the equivalent to doing:

In [4]: 
a = np.loadtxt("coastline.txt")
lon = a[:,0]
lat = a[:,1]

Now lets check the data:

In [5]: 
lat
Out[5]:
array([       nan,   7.664764,   7.699966, ..., -20.48083 , -20.210944,
       -20.023197])
In [6]: 
lon
Out[6]:
array([        nan,  134.729458,  134.738845, ...,   43.996118,
         44.151009,   44.315288])

For more complicated ascii files you can use the np.genfromtxt function. This command allows you to skip headers and specify data types and much more. For more info on reading in ascii files click here

Reading MATLAB and HDF5 files¶

To read in a MAT-file (saved in version earlier than 7.3), you can use the loadmat function in the scipy.io module:

In [7]: 
import scipy.io
M = scipy.io.loadmat("file.mat")
M
Out[7]:
{'__globals__': [],
 '__header__': 'MATLAB 5.0 MAT-file, Platform: MACI64, Created on: Wed Jul 23 16:54:04 2014',
 '__version__': '1.0',
 'random_array': array([[ 0.81472369,  0.0975404 ,  0.15761308,  0.14188634,  0.6557407 ],
       [ 0.90579194,  0.27849822,  0.97059278,  0.42176128,  0.03571168],
       [ 0.12698682,  0.54688152,  0.95716695,  0.91573553,  0.84912931],
       [ 0.91337586,  0.95750684,  0.48537565,  0.79220733,  0.93399325],
       [ 0.63235925,  0.96488854,  0.80028047,  0.95949243,  0.67873515]])}

.mat files are read in as dictionaries (see earlier discussion about dictionaries). Each variable is stored under a key named after the variable. Here, file.mat contains one variable, random_array. In the print out of M contains some meta-data. To give the random_array variable its own name, you would simply read into a new variable:

In [8]: 
rand_mat = M['random_array']

The scipy.io.loadmat only works for MAT-files saved in versions earlier than v-7.3. In MATLAB, you can specify the version when saving by using the version flag. Eg. save('file.mat','-v7.3').

MATLAB files saved in version -v7.3 or newer are in HDF5 format. Note that newer MATLAB releases use this version by default. To read these files into Python, you need to use the h5py module:

In [9]: 
import h5py
M2 = h5py.File('file_v73.mat')

In the above snippet, file_v73.mat is like the file.mat except that it was saved using the version -v7.3 flag in MATLAB. The M2 object is a special class called an HDF5 group. These groups work kinda like regular python dictionaries in that they have keys and data assigned to each key. To see the available keys, we would do:

In [10]: 
M2.keys()
Out[10]:
[u'random_array']

Here we see that there is only one key called random_array. You can ignore the u in front. It just means that the string is represented in unicode. To access the data stored under the key, we would do:

In [11]: 
M2['random_array'][:]
Out[11]:
array([[ 0.81472369,  0.90579194,  0.12698682,  0.91337586,  0.63235925],
       [ 0.0975404 ,  0.27849822,  0.54688152,  0.95750684,  0.96488854],
       [ 0.15761308,  0.97059278,  0.95716695,  0.48537565,  0.80028047],
       [ 0.14188634,  0.42176128,  0.91573553,  0.79220733,  0.95949243],
       [ 0.6557407 ,  0.03571168,  0.84912931,  0.93399325,  0.67873515]])

Note that the [:] at the end is necessary.

Warning about importing matlab arrays using h5py: The order of the dimensions of a matlab array will reverse after you import with h5py. This is because MATLAB stores data in column-major order while HDF (and Python) uses row-major order. So a matlab array with size 5x10x8 will be appear as a 8x10x5 numpy array in Python.

To learn more about h5py module take a look through the official documentation.

Reading Netcdf data¶

To work with netcdf files, you need the netcdf4 module. To demonstrate, we will import data from the 'cpc_precip_mon_mean.nc'. This file contains climatological precip data for the Bay of Bengal. To read in the file, we would do:

In [12]: 
import netCDF4 as nc4

f = nc4.Dataset('cpc_precip_mon_mean.nc','r')

The nc4.Dataset() functions works similar MATLAB's ncread function. The argument in parentheses is just the location of the file. The variable f is netcdf object. Data can be accessed using the following syntax: f.variables['variable_name'][:]. This requires that you know what the variable names are.

You might notice that f.variables['variable_name'][:] looks like a dictionary object - it is. The variable names are stored as dictionary keys. To display the key names, we can use the following loop:

In [13]: 
for name in f.variables:
    print name

lat
lon
time
precip

Now we know that the file contains lat, lon, time and precip data fields. Another way to explore a netcdf file is to use the print function:

In [14]: 
print(f)

This spits out all the meta data associated with the file. Towards the bottom, we again see that the file contains 4 variables but now we see their dimensions. Another way to inquire the dimensions of a variable is to do the following:

In [15]: 
f.variables['precip'].dimensions
Out[15]:
(u'time', u'lat', u'lon')

The output tells you that precip is a 3-d array with the first dimension representing time, the second dimension representing lat and the third dimension representing lon.

Now let's read the precip data into a new variable:

In [16]: 
precip = f.variables['precip'][:]

Note that the [:] at the end is needed. The same can be down for lat, lon and time:

In [17]: 
lat = f.variables['lat'][:]
lon = f.variables['lon'][:]

These variables are numpy arrays.

The precip data are explored further in the 'precip_demo.py' script. I included many comments, so hopefully it should be easy to follow. The script also demonstrates how to make plots from the data. To run the script sucessfully, the data files 'coastline.txt' and 'cpc_precip_mon_mean.nc' need to be in the same directory as precip_demo.py.

In [21]: 
%run precip_demo.py

Reading and saving data using Pickle¶

After reading data into Python, you might want to save it. There are ways of writing ascii and netcdf data to file but it might be more convenient to save data in format that is native to Python. Enter Pickle. https://wiki.python.org/moin/UsingPickle is the standard method of writing (serializing) Python objects, such as lists, arrays, dictionaries, to file. Here is an example of how to save an object:

In [19]: 
import pickle
import numpy as np

a = np.loadtxt("/Users/ewilson2011/python_work/plotting_demo/coastline.txt")
pickle.dump(a, open("coastline.p","wb"))

In the above example, we read in the coastline text file then save it as a pickle file using the dump command. The first argument of pickle.dump is the name of the object/variable that is to be saved. The second argument uses the open command to create an empty file into which pickle "dumps". The input "wb" means write-binary.

The general syntax for pickle.dump is pickle.dump(obj, file, protocol). The protocol parameter allows you to change the way in which pickle stores data. Setting protocol = pickle.HIGHEST_PROTOCOL or protocol = -1, tells pickle to use the fastest, most memory efficient option available. By using the highest protocol, you can save files that take up as much space as a netcdf or .mat file containing the same data.

To read in a previously saved pickle file, such as the fictional one we just created, you can use the following command:

In [20]: 
coastline_data = pickle.load(open("coastline.p","rb"))

We again make use of the open command. This time, the input "rb" means read-binary.
